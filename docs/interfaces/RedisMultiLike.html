    Interface RedisMultiLike

    A minimal, chainable interface describing a Redis MULTI transaction.

    Remarks

    Instances of this interface are created by calling IORedisLike.multi. Each method enqueues a command in the transaction and returns this so that you can fluently chain multiple operations before calling exec.

    The implementation is expected to mirror Redis semantics:

    • Commands are queued and not executed until exec() is called.
    • exec() resolves to an array of [error, reply] tuples, one per command, preserving order.

    Example

    const tx = client.multi()
      .set('user:1', JSON.stringify({ id: 1 }))
      .expire('user:1', 3600)
      .rpush('queue:jobs', 'job-1', 'job-2');

    const results = await tx.exec();
    // results: Array<[Error | null, any]>

    See

    IORedisLike for the corresponding client surface.

    exec

    • exec(): Promise<[Error | null, any][]>

      Execute the queued transaction (EXEC) and resolve results.

      Returns Promise<[Error | null, any][]>

      A promise resolving to an array of tuples [error, reply], one per queued command, in the same order.

      Remarks

      If the transaction is discarded or aborted by the server, behavior depends on the underlying client. Many clients reject the promise with an error.

      Example

      const results = await multi.exec();
      for (const [err, reply] of results) {
        if (err) console.error('Command failed:', err.message);
      }

      Throws

      If the underlying client fails to execute EXEC.
